Introduction

This API is designed using the REST architecture design, that means that every operation can be executed by making an HTTP request to the specified endpoint.

There are many ways to connect to the API, depending on the use case; but on the inside the connection relays on an HTTP request.

Connecting to this server

There is an instance of this project running on this server, which you can connect to, to access the API's resources.

Using an HTTP Client

The easiest way to connect to this API instance is using an HTTP client. This can be, either programatically (e.g. using the $.get method on jQuery), using a cli tool (e.g. curl) or a GUI client (e.g. PostMan or Insomnia).

Use this path to connect to the API: https://mexico-post.elotestudio.com/ and append the resource you want to access. The available endpoints are detailed on the Endpoints section.

Send an HTTP request to the desired resource's path, using the specified method (pretty much every single endpoint is exposed using GET).

Connecting another Loopback 4 app

With the introduction of Loopback 4, the framework developers implemented the OpenAPI spec. With this new addition, it is easier to connect external datasources to our projects. Just follow the instructions on how to add a datasource, more specifically the Datasource for REST service with OpenAPI specification topic.

When prompted for the spec file, enter https://mexico-post.elotestudio.com/explorer/openapi.json.

However, when using this approach, you will probably be required to copy some models into your own architecture; consider using the HTTP approach if you're not comfortable with this.

Connecting an OpenAPI-compatible framework

Depending on the specifics of your framework, you could need additional information. The OpenAPI spec file is available at https://mexico-post.elotestudio.com/explorer/openapi.json, but you can also use the API Explorer to help you get what you need.

Deploying your own server

The alternative is downloading a copy of the repository's code, and deploying and running the server on your own infrastructure; if that's what you're looking for.

There's really no reason to need to use this approach, other than security policies or maybe if you want to do some modifications to the code (which you're allowed to). The server is protected using the CloudFlare DNS and an SSL certificate, and the resources are perfectly accessible through this version. However, if you prefer so, you absolutely can do it.

Getting the code

Get a copy of the code by downloading a ZIP or cloning the GitHub repository: bjardon/mexico-post-api.

The code is released under the MIT license, so you're entitled to copy, modify and distribute; so long you give the proper credit to the author.

Configuring the server

Depending on the infrastructure, your config will vary. But you need to have a Node.js runtime installed and configured, and a port must be available for the HTTP server.

Make sure to install dependencies with npm.

Deploying the API

Once the API code has been downloaded and the Loopback app has been configured, the server can be deployed by starting the Node app.

Authentication

At the moment, no authentication method is required to access the resources. I don't know if this will change in the future; but in case it does, I'll let you know.
